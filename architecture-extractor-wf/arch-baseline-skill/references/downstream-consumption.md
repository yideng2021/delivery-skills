# 产物契约 + 下游消费方式（零耦合）

> 权威定义 arch-baseline 产出什么、下游如何消费。**核心原则：产物级弱耦合——本 skill 不感知任何特定下游，
> 下游按"产物契约"自行消费：存在即用、缺失则各自降级。**

---

## 一、产物契约（`.brownfield/`）

| 文件 | 内容 | 稳定性 |
|---|---|---|
| `00-overview.md` | 仓库画像 + 双索引状态 + 前置闸门留痕 + 时效约定 | 稳定 |
| `01-architecture.md` | 业务域地图 + 分层 + 主线 + 噪声剔除（人审） | 稳定（薄） |
| `02-business-flows.md` | 主线流程索引 + 入口锚点 + 活查询命令（人审） | 稳定（薄） |
| `05-retrieval-guide.md` | 活查询指南 + 下游消费映射 | 稳定 |

> 产物是**薄基线**；明细不固化，由下游按需活查询（CodeGraph/GitNexus）。

## 二、消费模式（与具体下游解耦）

任意下游可按下列通用场景消费，**无需知道 arch-baseline 的存在**——只需检测 `.brownfield/` 是否存在：

| 通用场景 | 消费方式 |
|---|---|
| **现状盘点**（既有能力/模块触达） | 读 `01` 业务域 + `02` 锚点 → `gitnexus context` / `codegraph_impact` 确认 |
| **复用决策取证**（扩展 vs 新建） | `cypher MEMBER_OF` / `gitnexus context` 看责任归属 → `codegraph_impact` 量化影响面 |
| **影响驱动拆分** | `codegraph_impact` / `gitnexus detect_changes` 把爆炸半径换算成任务/边界 |
| **架构学习/导览** | 交 `arch-guide`（同 wf）展开为厚导览 |

## 三、解耦纪律

- **零命令耦合**：本 skill 不被任何下游 import/调用；下游也不得把本 skill 列为硬前置依赖。
- **存在即用、缺失降级**：下游检测 `.brownfield/` 在则消费、不在则各自回退（如裸代码搜索），不报错、不阻断。
- **不命名约束**：下游文档若引用本产物，引用的是**产物路径契约**（`.brownfield/...`），而非本 skill 名。
- 本 skill **不修改** 目标仓源码、不写任何下游的文件。

## 1. 目录结构规范 (Standard Directory Layout)

> 基于 `{language/framework}` 的标准分层结构：

```
project-root/
├── src/
│   ├── api/                 # 接口定义层 (Controller / Handler / Router)
│   ├── service/             # 业务逻辑层 (Service / UseCase)
│   ├── domain/              # 领域模型层 (Entity / ValueObject / Aggregate)
│   ├── infrastructure/      # 基础设施层 (Repository / Gateway / Config)
│   │   ├── persistence/     # 持久化实现
│   │   ├── external/        # 外部服务调用
│   │   └── config/          # 配置类
│   └── common/              # 公共组件
│       ├── constants/       # 常量定义
│       ├── enums/           # 枚举类
│       ├── utils/           # 工具类
│       └── exception/       # 异常定义
├── tests/                   # 测试代码 (与 src 目录结构镜像)
├── docs/                    # 项目文档
├── scripts/                 # 构建、部署、数据迁移脚本
└── config/                  # 外部化配置文件
```

**分层依赖原则**：`api → service → domain ← infrastructure`

- `domain` 层不依赖任何外部层（最内层）
- `infrastructure` 实现 `domain` 层定义的接口（依赖倒置）
- `api` 层仅做参数校验和协议转换，禁止包含业务逻辑

## 2. 统一错误码体系 (Error Code System)

### 2.1 编码格式

```
格式: A-BB-CCC

A    → 错误来源 (1位)
BB   → 模块编号 (2位)
CCC  → 具体错误编号 (3位)
```

| 位   | 含义     | 取值说明                                            |
| ---- | -------- | --------------------------------------------------- |
| A    | 错误来源 | `1`: 系统级, `2`: 业务级, `3`: 第三方服务           |
| BB   | 模块编号 | `00`: 通用, `01`: 用户, `02`: 订单, `03`: 支付...   |
| CCC  | 错误编号 | 模块内唯一递增                                       |

### 2.2 预置错误码

| 错误码  | 描述                 | HTTP 状态码 | 建议处理           |
| ------- | -------------------- | ----------- | ------------------ |
| 200000  | 操作成功             | 200         | -                  |
| 100001  | 系统内部错误         | 500         | 告警 + 重试        |
| 100002  | 服务不可用           | 503         | 降级 + 告警        |
| 100003  | 请求超时             | 504         | 重试               |
| 200100  | 参数校验不通过       | 400         | 返回详细字段错误   |
| 200101  | 资源不存在           | 404         | 提示用户           |
| 200102  | 权限不足             | 403         | 提示并记录         |
| 200103  | 重复操作             | 409         | 幂等处理           |
| 300001  | 第三方服务调用失败   | 502         | 重试 + 降级        |

## 3. Git 提交规范 (Commit Convention)

### 3.1 提交信息格式

```
<type>(<scope>): <subject>

[可选 body]

[可选 footer]
```

### 3.2 Type 类型

| type       | 说明                         | 示例                                |
| ---------- | ---------------------------- | ----------------------------------- |
| `feat`     | 新功能                       | `feat(user): 新增手机号登录`        |
| `fix`      | Bug 修复                     | `fix(order): 修复金额计算精度`      |
| `docs`     | 文档变更                     | `docs(readme): 更新部署说明`        |
| `style`    | 代码格式（不影响逻辑）       | `style: 格式化代码缩进`             |
| `refactor` | 重构（非新功能、非修复）     | `refactor(auth): 重构鉴权中间件`    |
| `perf`     | 性能优化                     | `perf(query): 优化列表查询索引`     |
| `test`     | 测试相关                     | `test(user): 补充用户注册单元测试`  |
| `chore`    | 构建/工具变动                | `chore: 升级 Node.js 至 20.x`      |
| `ci`       | CI/CD 配置变更               | `ci: 新增 staging 部署流水线`       |

### 3.3 分支策略

| 分支          | 用途                 | 命名规范                        |
| ------------- | -------------------- | ------------------------------- |
| `main`        | 生产主干             | -                               |
| `develop`     | 开发集成             | -                               |
| `feature/*`   | 功能开发             | `feature/user-login`            |
| `bugfix/*`    | Bug 修复             | `bugfix/fix-amount-precision`   |
| `release/*`   | 发布分支             | `release/v1.2.0`                |
| `hotfix/*`    | 紧急修复             | `hotfix/fix-critical-crash`     |

## 4. 日志规范 (Logging Standards)

### 4.1 日志级别使用

| 级别    | 使用场景                                 |
| ------- | ---------------------------------------- |
| `ERROR` | 系统异常、不可恢复的错误                 |
| `WARN`  | 可恢复的异常、需要关注的降级             |
| `INFO`  | 重要业务节点（入口、出口、状态变更）     |
| `DEBUG` | 开发调试信息（生产环境默认关闭）         |

### 4.2 日志格式

```
[时间] [级别] [TraceID] [类名.方法名] - 消息内容
```

### 4.3 日志禁忌
- ❌ 打印敏感信息（密码、Token、身份证号）
- ❌ 在循环中打印 INFO 级别日志
- ❌ 使用 `System.out.println` / `console.log` 替代日志框架

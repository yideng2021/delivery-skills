### 3、界面流程节点详细

##### 节点 1(示例)：基本信息编辑（Step 1）

- **节点描述**：需求计划编辑第一步——填写/修改基本信息（采购类型、合同方式等），暂存后进入下一步
- **关联 API**：
  - `GET /user/api/org/getLevelTwoOrgByOrgCode/61012` — 获取二级组织信息 `[外部:user]`
  - `GET /requirement/api/schemeRequirement/{id}` — 获取需求计划详情
  - `GET /requirement/api/strategyProjectPlan/allPageList` — 查询策略项目计划列表
  - `GET /contract/api/contract/getPurchaseOrgContractPage` — 查询采购组织合同分页 `[外部:contract]`
  - `GET /requirement/api/schemeRequirementAttachment/getListByRequireId/{id}` — 获取附件列表
  - `GET /infrastructure/api/attachment/list/{type}/objectId/{id}` — 获取附件详情 `[外部:infrastructure]`
  - `GET /requirement/api/strategyProject/pageListByOrgCode` — 按组织查询项目信息库
  - `POST /requirement/api/schemeRequirement/addOrUpdate` — 暂存需求计划基本信息
- **前端关联代码**：
  - 路径：`src/views/requirement/purchaseRequire/requireScheme/DemandPlanSubmission/modules/stepModules/BaseInfo/Index.vue`、`src/api/schemeRequire.js`
  - 调用逻辑：进入编辑页时调用 `getSchemeRequirementById()` 加载详情，修改后调用 `addOrUpdateSchemeRequirement()` 暂存
- **后端关联代码**：
  - 类：`SchemeRequirementController.getById()` / `SchemeRequirementController.addOrUpdate()`
  - 类：`SchemeRequirementAttachmentController.getListByRequireId()`
  - 类：`StrategyProjectPlanController.allPageList()` / `StrategyProjectController.pageListByOrgCode()`
  - 接口逻辑：按 ID 查询需求计划实体 → 前端渲染表单 → 用户修改后调用 addOrUpdate 保存（insert 或 update 取决于是否有 id）
- **上下文关联**：暂存成功后用户点击「下一步」进入供应商管理步骤

##### 节点 5：供应商管理（Step 2）

- **节点描述**：管理需求计划的推荐供应商——删除已有供应商、新增供应商（弹窗搜索选择）、执行合规风险检测，循环操作直至满意
- **关联 API**：
  - `GET /requirement/api/schemeRequirementDetail/getByRequireId/{id}` — 获取需求计划采购详情
  - `GET /requirement/api/schemeRequirement/{id}` — 刷新需求计划信息
  - `GET /requirement/api/schemeRequirementSupplier/getRequireSupplierList/{id}` — 获取推荐供应商列表
  - `GET /requirement/api/purchaseApplication/getClassifyReportList` — 获取品类策略报告列表
  - `DELETE /requirement/api/schemeRequirementSupplier/deleteById/{id}` — 删除推荐供应商
  - `GET /classifymanage/api/supplier/archives/api/supplier/archives/list` — 搜索供应商档案 `[外部:classifymanage]`
  - `POST /requirement/api/schemeRequirementSupplier/getSupplierBySupplierIdList/{id}` — 校验供应商是否已存在
  - `POST /requirement/api/schemeRequirementSupplier/insertList` — 批量新增推荐供应商
  - `POST /classifymanage/api/supplierrisk/getCheckResult` — 合规风险检测 `[外部:classifymanage]`
  - `GET /requirement/api/schemeRequirementSupplier/checkHighSupplier/{id}` — 检查高风险供应商
- **前端关联代码**：
  - 路径：`src/views/requirement/purchaseRequire/requireScheme/DemandPlanSubmission/modules/stepModules/ProcurementInformation/Index.vue`、`src/views/.../modules/SelectSupplierModal.vue`、`src/api/schemeRequire.js`
  - 调用逻辑：加载供应商列表 `getRequireSupplierList()`；删除 `deleteRequireSupplier()`；新增流程：打开弹窗搜索供应商(`classifymanage`) → 选中 → `getSupplierBySupplierIdList()` 校验 → `insertRequireSupplierList()` 保存 → `getCheckResult()` 风控检测 → 再次 `insertList` 更新风险等级 → 刷新列表
- **后端关联代码**：
  - 类：`SchemeRequirementSupplierController.getRequireSupplierList()` / `deleteById()` / `getSupplierBySupplierIdList()` / `insertList()` / `checkHighSupplier()`
  - 类：`PurchaseApplicationController.getClassifyReportList()`
  - 接口逻辑：供应商 CRUD + 风险等级校验；每次添加供应商触发循环：**保存 → 高风险校验 → 风控检测(classifymanage) → 更新风险等级(insertList) → 刷新列表**
- **上下文关联**：供应商确认完毕后，点击「下一步」进入提交确认步骤

### 节点 6：提交确认与审批发起（Step 3）

- **节点描述**：最终确认需求计划信息、删除多余供应商、保存详情、校验供应商风险，选择审批流程并发起审批
- **关联 API**：
  - `DELETE /requirement/api/schemeRequirementSupplier/deleteById/{id}` — 删除多余供应商
  - `POST /requirement/api/schemeRequirementDetail/add` — 保存需求计划详情
  - `POST /requirement/api/schemeRequirement/addOrUpdate` — 保存需求计划
  - `GET /requirement/api/schemeRequirementSupplier/checkHighSupplier/{id}` — 高风险供应商检查
  - `POST /classifymanage/api/supplierrisk/getCheckResult` — 风控检测 `[外部:classifymanage]`
  - `POST /requirement/api/schemeRequirementSupplier/insertList` —更新供应商风险等级
  - `GET /requirement/api/schemeRequirementSupplier/getRequireSupplierList/{id}` — 刷新供应商列表
  - `GET /requirement/api/schemeRequirement/{id}` — 获取最新需求计划信息
  - `GET /requirement/api/schemeRequirementDetail/getByRequireId/{id}` — 获取需求计划详情
  - `GET /requirement/api/schemeRequirementSupplier/checkSupplierRisk/{id}` — 校验供应商整体风险
  - `GET /requirement/api/schemeRequirement/isToOneLevel/{id}` — 判断是否直推一级统筹
  - `GET /infrastructure/api/purchasingProcess/getPurchasingProcessByBusinessTypeAndObjectId` — 获取审批流程配置 `[外部:infrastructure]`
  - `POST /requirement/api/schemeRequirement/start` — 发起审批流程（继承自 `BaseFlowableController`）
- **前端关联代码**：
  - 路径：`src/views/requirement/purchaseRequire/requireScheme/DemandPlanSubmission/modules/stepModules/ProcurementInformation/Index.vue`（「确认提交」按钮）、`src/api/schemeRequire.js`
  - 调用逻辑：点击「确认提交」→ 保存 detail + addOrUpdate → 校验供应商风险 → 判断 isToOneLevel → 获取审批流程 → 弹出流程选择 iframe → 用户选择流程后点击「发起审批」→ 调用 `startFlowSchemeRequirementChange()`
- **后端关联代码**：
  - 类：`SchemeRequirementController.start()` — 继承自 `BaseFlowableController.start()`，回调 `SchemeRequirementService.startBefore()` + `startAfter()`
  - 类：`SchemeRequirementDetailController.add()` → `SchemeRequirementDetailService.addOrUpdate()`
  - 类：`SchemeRequirementSupplierController.checkSupplierRisk()` / `checkHighSupplier()`
  - 接口逻辑：保存表单 → 风险校验通过 → `start` 发起工作流（设置 flowStatus=IN_THE_PROCESS, requireStatus=IN_REVIEW）
- **上下文关联**：审批发起后系统自动返回列表页，需求状态变为「审核中」

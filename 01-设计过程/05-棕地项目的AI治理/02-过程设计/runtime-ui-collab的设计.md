# runtime-ui-collab Skill 设计方案

> 定位：基于 runtime-flow-mapper 和 runtime-spec 产物，通过人机协同逐界面确认，输出高质量界面业务梳理文档。
> 上游 Skill：runtime-capture → runtime-flow-mapper → runtime-spec-mapper → **runtime-ui-collab**

---

## 一、触发与输入

**用户提供**：
- `flow-map` 文件路径（必须）
- `spec` 文件路径（必须）

**启动后**：AI 读取两份文件，根据内容完整性动态决定是否追问前置澄清问题（如角色歧义、界面边界不清等）。内容清晰则直接进入节点0。

---

## 二、执行阶段

### 节点 0：界面范围确认

1. AI 读取 flow-map 节点清单，推断界面映射（节点 → UI-xx）
2. 展示界面清单表格（含界面ID / 名称 / 类型 / 对应 flow-map 节点）
3. 用户确认或修正界面范围
4. 确认后进入逐界面协同阶段

**表格格式**：

| 界面ID | 界面名称 | 类型 | 对应节点 | 触发入口 |
|--------|---------|------|---------|---------|
| UI-01 | ... | 独立/合并/子界面/多视角 | 节点N | ... |

---

### 节点 1~N：逐界面人机协同（每个界面循环）

**Step 1 — 草稿生成**

AI 基于 flow-map + spec 生成当前界面完整草稿：
- 业务介绍（是什么/谁用/能做什么/关键约束）
- 上下文说明（前置/后置/数据/权限/状态五维度）
- 功能地图（骨架树形 + 每个功能点详细描述）
- 界面流程图（Mermaid）

草稿中所有推断内容用 `~推断` / `~待确认` 标注。

---

**Step 2 — 确认菜单**

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  UI-01 · [界面名称] — 草稿已生成
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
请选择操作：
  [1] 全部确认，进入下一界面
  [2] 修正业务介绍 / 上下文
  [3] 修正功能地图
  [4] 补充遗漏功能
  [5] 现在输出此界面文档
  [6] 跳过此界面
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

---

**Step 3 — 推断点主动追问**

每次展示草稿后，AI 自动识别草稿中的 `~推断` / `~待确认` 点，逐一追问。

问题呈现规则：
- **复杂推断**（业务规则、分支逻辑、不确定的 API）→ 卡片式

```
┌─ ❓ 待确认 ──────────────────────────────────────────┐
│ 问题：[功能X] 的触发方式                             │
│ 我的推断：用户点击列表行的「编辑」按钮               │
│ 推断依据：flow-map 节点4 描述 + 前端组件命名         │
│ 推荐答案：是                                         │
│ 可选：[是] [否，实际是______] [不确定，保留标注]     │
└──────────────────────────────────────────────────────┘
```

- **简单确认**（字段名、操作文案等） → 对话式直接问

---

**Step 4 — 用户回答 → 更新草稿**

AI 根据用户回答更新草稿内容和确信度标注，完成后重新展示确认菜单，直至用户选择「确认」或「跳过」。

---

### 收尾阶段

所有界面处理完毕后：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  所有界面协同完成
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  已确认：UI-01 / UI-02
  已跳过：UI-03
  请选择：
  [1] 统一输出所有已确认界面文档
  [2] 查看并输出单个界面
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

---

## 三、输出

**目录**：AI 扫描工程现有 `docs/prd/` 目录结构，推断合适输出路径，展示给用户确认后写入。

**文件命名**：`{界面名称}.md`（每个界面一个文件）

**文档结构**（每个界面文件）：
```
# {界面名称} — 界面业务梳理
## 一、界面业务描述
  1.1 业务介绍
  1.2 截图（占位）
  1.3 上下文说明
## 二、界面功能地图
  2.1 功能骨架总览
  2.2 功能详细描述
## 三、界面流程图
```

---

## 四、确信度标注规范

| 标注 | 含义 |
|------|------|
| `[runtime+码]` | 运行时录制 + 代码双重验证（最高） |
| `[码]` | 代码直接验证 |
| `[人工]` | 人工确认 |
| `~推断` | AI 推断，未直接验证 |
| `~待确认` | 存疑，等人工确认 |

---

## 五、与上游 Skill 的关系

```
runtime-capture        ← 采集运行时操作与 API 时序
    ↓
runtime-flow-mapper    ← 生成流程精准地图（flow-map）
    ↓
runtime-spec-mapper    ← 生成现状业务规约（spec）
    ↓
runtime-ui-collab      ← 人机协同，逐界面梳理输出业务文档（本 Skill）
```

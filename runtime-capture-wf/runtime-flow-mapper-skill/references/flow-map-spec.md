# 精准地图产物规格（`{name}_flow-map.md`）

## 六个板块

### 1. 业务流程简介
2-3 句：业务场景 + 核心价值。

### 2. 流程节点清单（按时序）
每个节点：
```
### 节点 N：[节点名称]
- 节点描述：1 句业务含义
- 关联 API：
  - `METHOD /api/xxx` — 用途简述
- 前端关联代码：
  - 路径：`src/xxx/Xxx.vue` / `src/api/xxx.js`
  - 调用逻辑（1-2 句，结合本节点上下文）
- 后端关联代码：
  - 类：`com.xxx.XxxController` / `XxxService`
  - 接口逻辑（1-2 句：校验→查询→转换→持久化等）
- 上下文关联：与上/下节点的数据流转、状态依赖
```
> 命名用业务语义（非技术术语）；逻辑须结合上下文串联，不孤立描述。

### 3. 界面-API-Handler 三证对齐表（速查总表）
| 界面操作 | HAR API | 前端组件 | 前端 API 函数 | 后端 Handler |
|---------|---------|---------|--------------|-------------|
未确证项标 `~inferred`。

### 4. 外部系统调用
| 外部系统 | API 路径 | 用途 |
（path 前缀非本服务的接口归此，不落本仓代码。）

### 5. 状态流转
状态机（流程状态 / 处理状态等），单样本推断标 `~inferred`。

### 6. 流程图（mermaid）
节点 = 界面/子过程；边 = 触发的接口；与节点清单**同源一致**。
```mermaid
flowchart TD
  ...
```

## 校验清单（产出前自检）

- [ ] **真实**：每条 API 都能在 `{name}_api_requests.txt` 找到（不杜撰）。
- [ ] **归属**：每条接口落到某节点；归属依据 session_log 修正后 URL。
- [ ] **三证**：确证项三证交叉；单证项已标 `~inferred`/`?`。
- [ ] **代码锚**：每个前后端代码锚有检索依据（codegraph/gitnexus/grep）。
- [ ] **外部系统**：跨服务接口单列，未强行落本仓。
- [ ] **图文一致**：mermaid 节点/接口集合与节点清单一致。
- [ ] **上下文**：节点逻辑结合前后节点串联，非孤立。

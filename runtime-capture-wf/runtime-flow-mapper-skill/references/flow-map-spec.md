# 精准地图产物规格（`{name}_flow-map.md`）

> 主产物结构 + 校验清单。骨架见 [`../templates/flow-map.md`](../templates/flow-map.md)。
> 形态范本（已验证）：`01-设计过程/03-testing/playwright录制/runtime-flows/flow-code-map.md`。

## 六个板块

### 1. 业务流程简介
2-3 句：业务场景 + 核心价值。

### 2. 流程节点清单（按时序）
每个节点：
```
### 节点 N：[节点名称]
- 节点描述：1 句业务含义
- 关联 API：
  - `METHOD /api/xxx` — 用途简述
  - `METHOD /classifymanage/api/xxx` — 用途 `[外部:classifymanage] 本地工程无源码，来源：共享组件/微前端`
  > 节点内出现外部接口必须标 `[外部:xxx]` + 源码缺失结论（见 correlation §4），不得悬空。
  > 接口呈现循环/因果链时，在节点描述里说明触发条件与循环逻辑（见 correlation §7）。
- 前端关联代码：
  - 路径：`src/xxx/Xxx.vue` / `src/api/xxx.js`
  - 调用逻辑（1-2 句，结合本节点上下文）
- 后端关联代码：
  - 类：`com.xxx.XxxController` / `XxxService`
  - 接口逻辑（1-2 句：校验→查询→转换→持久化等）
- 上下文关联：与上/下节点的数据流转、状态依赖
```
> 命名用业务语义（非技术术语）；逻辑须结合上下文串联，不孤立描述。

### 3. 界面-API-Handler 三证对齐表（速查总表）
| 界面操作 | HAR API | 前端组件 | 前端 API 函数 | 后端 Handler |
|---------|---------|---------|--------------|-------------|
**覆盖率要求**：须覆盖**全部本服务（非外部）去重接口**，不得只挑核心接口。
可分「核心接口」「辅助接口」两组，辅助接口以精简行列出，但**不得遗漏**。
未确证项标 `~inferred`；外部接口不进本表（归板块 4）。

### 4. 外部系统调用
| 外部系统 | API 路径 | 用途 | 前端调用来源 |
|---------|---------|------|------------|
（path 前缀非本服务、且前端 grep 0 匹配的接口归此。）
**前端调用来源** 填：`本工程 src/api/xx.js` 或 `未在本工程找到，疑为共享组件/微前端注入`（见 correlation §4）。

### 5. 状态流转
状态机（流程状态 / 处理状态等），单样本推断标 `~inferred`。

### 6. 流程图（mermaid）
节点 = 界面/子过程；边 = 触发的接口；与节点清单**同源一致**。
**禁止任何样式定义**（`classDef` / `style` / `class`）——只保留结构，渲染环境会拒绝样式元素。
```mermaid
flowchart TD
  ...
```

## 校验清单（产出前自检）

- [ ] **真实**：每条 API 都能在 `{name}_api_requests.txt` 找到（不杜撰）。
- [ ] **归属**：每条接口落到某节点；归属依据 session_log 修正后 URL。
- [ ] **三证**：确证项三证交叉；单证项已标 `~inferred`/`?`；纯运行时存在项标 `~external:unreachable`。
- [ ] **代码锚**：每个前后端代码锚有检索依据（codegraph/gitnexus/grep）；前端组件路径经工具验证存在，否则标 `~inferred`。
- [ ] **覆盖率**：三证对齐表覆盖全部本服务去重接口（可分核心/辅助），无遗漏。
- [ ] **外部系统**：跨服务接口经前端 grep 验证后单列，填「前端调用来源」；节点内引用外部接口已标 `[外部:xxx]`+源码缺失结论，无悬空。
- [ ] **节点切分**：`[NAV:*]` 与归属 URL 域名变化均已切节点（跨系统跳转未漏）。
- [ ] **时序**：循环/因果链接口已在节点描述中说明触发条件，非平铺。
- [ ] **图文一致**：mermaid 节点/接口集合与节点清单一致；**无 classDef/style 样式**。
- [ ] **上下文**：节点逻辑结合前后节点串联，非孤立。

# 输入前置人工确认（强制·首步）

> 录制前先收集输入、**回显实际路径请用户确认**，再检查运行时是否在位。
> 让本 skill 成为**无门槛工具服务**：用户只需给 3 个输入，环境由 ① 自动准备。

## A. 收集输入（缺即停，向用户要）

| 输入 | 用途 | 示例 |
|------|------|------|
| **工作目录** | 一次确定，后续所有产物都在其下 | `D:\runtime-capture` 或 `docs/business-processes/采购/runtime-flows` |
| **业务流程名** `{name}` | 产物前缀 + 子文件夹名 + 闭环边界 | `requirement-plan-filing` |
| **录制起始 URL** | 录制入口 | `https://req.xxx.com/#/login` |

> `{name}` 用 kebab-case。被测服务须已启动、可访问、有测试账号。

## B. 人工确认环节（回显实际路径，等用户点头）

把解析后的**实际落点**回显给用户，明确将创建/写入哪些路径，确认后才动手：

```
【请确认本次运行配置】
  工作目录:     {工作目录}
  执行环境:     {工作目录}/runtime_environment/      （脚本+依赖，自动准备）
  本次产物目录: {工作目录}/{name}/                    （5 件产物落此）
  录制起始 URL: {url}
  业务闭环边界: {与用户口头确认：含 列表/新增/详情/删除 等哪些子过程}

确认无误请回复"确认"；需调整请指出。
```

> 业务闭环边界必须**录前与用户对齐**（不变量 1）：不录碎片、不无限扩张。

## C. 依赖检查（只检查，缺则告知命令；skill 不代装）

| 项 | 检查 | 缺失处置 |
|----|------|----------|
| Node ≥ 18 | `node -v` | 告知用户安装 Node；BLOCK |
| Python 3 | `python --version` | 告知用户安装 Python；BLOCK |
| chromium 内核 | 工序 ① `setup.mjs` 末尾自动检测（查 `ms-playwright/chromium-*`） | 缺则 setup 打印命令 `npx playwright install chromium` 告知用户手动装；BLOCK |

> **唯一自动的安装** = 工序 ① 在 `runtime_environment/` 内的 `npm install`（拉取 `@playwright/test`，幂等、先检查再执行）。
> Node / Python / **浏览器内核**均**只检查 + 告知命令，由用户手动安装**——较重安装人工执行更合适，也简化 skill 复杂度。

## 三态裁决

| 态 | 条件 | 动作 |
|----|------|------|
| **PASS** | 输入齐全 + 用户已确认 + Node/Python 在位 | 进入 ① 环境准备 |
| **CONFIRM** | 输入齐但路径/边界待用户拍板 | 回显 B 块，等"确认" |
| **BLOCK** | 缺输入 / Node 或 Python 缺失 | 列全清单转人工，补齐后重新触发，**不轮询、不臆断默认值** |

## 待人工处理清单（BLOCK 时输出此格式）

```
【前置确认清单 — 待人工处理（请一次性补齐，我不重复轮询）】

运行时
  [✓] Node v20.11
  [✗] Python 未安装        → 请安装 Python 3（解析脚本需要）

待提供输入
  [ ] 工作目录：________
  [ ] 业务流程名（kebab-case）：________
  [ ] 录制起始 URL：________
  [ ] 被测服务是否已启动 + 测试账号：________

裁决：BLOCK（Python 缺失 + 输入未提供）。补齐后重新触发本 skill。
```

## 纪律

- **人工确认不可跳过**：未得用户对路径/边界的确认，不进入 ①。
- **只一处自动**：仅 `runtime_environment` 的 `npm install` 自动且幂等；Node/Python/chromium 一律只检查 + 告知命令，用户自行装。
- **不臆断**：工作目录/`{name}`/URL 缺失即问，不擅自取默认。

# 运行时业务流程录制与生成 — 提示词模板  
  
---  
  
## 阶段一：录制运行时业务流（runtime-capture）  
  
```  
录制运行时业务流：  
  
- 业务流程名：{中文名称，如"需求计划填报流程"}  
- 起始URL：{业务入口页面URL}  
- 工作目录：{录制产物存放路径}  
- 前端工程目录：{前端源码绝对路径}  
- 后端工程目录：{后端源码绝对路径}  
```  
  
---  
  
## 阶段二：生成运行时业务流程精准地图（runtime-flow-mapper）  
  
```  
生成运行时业务流程精准地图：  
  
- 业务流程名：{与录制阶段同名}  
- 工作目录：{录制产物所在路径}  
- 前端工程目录：{前端源码绝对路径}  
- 后端工程目录：{后端源码绝对路径}  
```  
  
---  
  
## 使用要点  
  
1. **两阶段分开执行**：先完成录制采集，确认产物齐全后再触发地图生成  
2. **工程路径务必准确**：路径错误会导致代码落锚全部失败  
3. **录制时完整操作一遍业务闭环**：从进入页面到最终提交/完成，覆盖核心分支  
4. **录制完成后检查产物**：确认 `_session_log.txt`、`_playwright_records.ts`、`_api_requests.txt`、`_api_details.txt` 四件齐全  
5. **外部系统接口**：若涉及跨服务调用（微前端/共享组件），地图会标注为外部系统，如需深入落锚需提供对应服务的工程目录

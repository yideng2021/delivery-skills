# E2E 测试审查清单 (E2E Testing Checklist)

> 在 **阶段 5: 报告生成** 时逐项核对。
> 全部通过方可生成最终报告，未通过项需标注原因。
> 本清单持续沉淀 E2E 测试领域经验，可随项目迭代扩充。

---

## 1. 测试覆盖完整性 (Coverage Completeness)

- [ ] **模块覆盖**：测试策略中列出的所有模块是否都已执行测试？
- [ ] **正向用例**：每个模块的核心正向流程（CRUD）是否已覆盖？
- [ ] **逆向用例**：每个模块的必填校验、格式校验、边界值校验是否已覆盖？
- [ ] **页面加载**：每个被测页面是否有初始加载状态的验证和截图？
- [ ] **数据依赖**：模块间的数据依赖关系是否按正确顺序执行？
- [ ] **取消/关闭操作**：弹窗取消、页面返回等"放弃"操作是否已验证？

## 2. 测试执行质量 (Execution Quality)

- [ ] **真实执行**：所有测试结果是否来自 `playwright-cli` 真实浏览器操作（非模拟）？
- [ ] **截图完整**：每个关键步骤（页面加载、表单填写、提交结果、错误提示）是否都有截图？
- [ ] **截图命名**：截图是否遵循 `{两位全局序号}_{模块简称}_{场景}.png` 命名规范？
- [ ] **截图可读**：截图分辨率是否足够（推荐 1920x1080），关键信息是否清晰可见？
- [ ] **逐步执行**：是否每条 `playwright-cli` 命令都独立执行并检查了输出？
- [ ] **snapshot 管理**：是否在每次 DOM 变化后都重新执行了 `snapshot`？

## 3. Bug 记录质量 (Bug Recording Quality)

- [ ] **如实记录**：是否所有发现的问题都已记录，无遗漏或美化？
- [ ] **Bug 分级**：每个 Bug 是否标注了严重级别（Critical / Major / Minor / Suggestion）？
- [ ] **复现步骤**：每个 Bug 是否包含完整的复现步骤？
- [ ] **截图证据**：每个 Bug 是否配有对应的截图作为证据？
- [ ] **修复建议**：每个 Bug 是否提供了修复建议并标注了回溯目标（前端/后端/设计）？

## 4. 回归脚本质量 (Regression Script Quality)

- [ ] **脚本完整**：是否每个测试模块都有对应的 `.sh` 脚本？
- [ ] **ref 注释**：脚本中每条 `click/fill/select` 后是否标注了 ref 含义？
- [ ] **snapshot 标记**：脚本中每条 `snapshot` 后是否标注了 `★ 此处刷新ref` 提示？
- [ ] **环境参数化**：脚本顶部是否定义了 `BASE_URL` 变量？
- [ ] **执行顺序**：`run_all.sh` 是否按模块依赖顺序串联？
- [ ] **可执行性**：脚本是否有可执行权限（`chmod +x`）？

## 5. 报告质量 (Report Quality)

- [ ] **五板块齐全**：报告是否包含 测试概览 / 业务理解 / 执行明细 / Bug清单 / 截图画廊 五个板块？
- [ ] **数据准确**：报告中的用例统计（总数/通过/失败/通过率）是否与实际执行一致？
- [ ] **截图内嵌**：截图是否以 Base64 格式内嵌到 HTML 中（自包含，无外部依赖）？
- [ ] **样式一致**：报告是否使用了统一的卡片式 / 斑马纹表格 / 颜色徽章样式？
- [ ] **可导航**：报告是否有顶部锚点导航（快速跳转到各板块）？
- [ ] **技术栈标注**：报告是否标注了被测系统的技术栈信息（来自 agent.md）？

## 6. 闭环追溯性 (Traceability)

- [ ] **用例 → AC 映射**：每个测试用例是否标注了对应的验收标准 (AC) 编号（若有）？
- [ ] **Bug → 上游映射**：每个 Bug 是否标注了应回溯至哪个上游 Skill（spec-frontend-dev / spec-backend-dev / spec-design）？
- [ ] **无私自修复**：是否确认测试过程中未修改任何业务代码（只记录问题，不修复）？
- [ ] **数据清理**：测试产生的临时数据是否已标注（方便后续清理或不影响其他测试）？

## 7. 环境与兼容性 (Environment & Compatibility)

- [ ] **环境信息**：报告中是否记录了测试环境（URL、浏览器版本、操作系统）？
- [ ] **服务可用**：测试执行期间被测服务是否始终可用（无中断）？
- [ ] **分辨率一致**：所有截图是否使用了统一的浏览器分辨率？
- [ ] **时间记录**：报告中是否记录了测试执行时间（开始/结束/总耗时）？

# 代码深探法（核心增值）

> 本 skill 的立身之本：**以前后端代码为第一依据**，把 flow-map 的"接口级落锚"深化为"业务级规约"。
> flow-map 给「在哪」（哪个节点、哪个 handler），代码给「是什么」（实体、规则、完整状态机）。
> 检索工具口径复用 [`../../runtime-flow-mapper-skill/references/search-guide.md`](../../runtime-flow-mapper-skill/references/search-guide.md)，本文只讲**深探链路与抽取规约**。

## 1. 深探入口：从 flow-map 的代码锚出发

flow-map 三证对齐表已给出每个节点的**后端 handler** 与**前端 API 函数/组件**。本 skill **不重做落锚**，直接接过这些锚往下读。

> 锚为 `~unresolved`/`~inferred` 的接口：深探时优先尝试补一次（继承路由按 search-guide 继承链分支），补不到则该项规约整体标 `~inferred`。

## 2. 深探链路（有界，到此为止）

```
handler（flow-map 已锚）
  → codegraph_explore(handler)            读方法体主逻辑
  → codegraph_callees(handler)            顺链进 Service
  → Service 主逻辑                         抽：业务规则 INV（校验/分支/抛错）
  → 入参/出参 DTO·VO（codegraph_explore）  抽：业务实体 + 属性清单
  → 枚举 / 常量类                          抽：完整状态机（补 flow-map 单样本盲区）
前端：
  → flow-map 锚的组件 / 校验规则（rules / validator）  抽：字段必填·格式·前端约束
```

**边界（不变量 6）**：**不下 DAO/SQL、不展开完整调用链**。Service 主逻辑 + DTO + 枚举 + 前端校验即足以梳理业务级现状；再深就是技术实现细节，噪声大、业务语言翻译难，超出现状规约范畴。

## 3. 抽取规约：从代码到业务概念

| 抽取目标 | 代码来源 | 落到规约 |
|----------|----------|----------|
| **业务实体 + 属性** | 入参/出参 DTO/VO、Entity 字段 | L2 业务实体（属性用业务语义，字段名进溯源注） |
| **业务规则 INV** | Service 的校验（`if…throw`/断言/`@Valid`）、分支条件 | L2 INV（陈述为现状约束） |
| **完整状态机** | 状态枚举 + 流转方法（`status=` 赋值处） | L2 业务流转（枚举给全集，flow-map 给已跑到的迁移） |
| **前端字段约束** | 表单 rules / validator / 必填标记 | L2 INV（前端侧）或实体属性约束 |
| **用户故事** | flow-map 节点语义 + handler 业务动作 | L1 US |

## 4. 现状 ≠ 规范（最危险的纪律，呼应不变量 2）

代码里的一条 `if (x > 100) throw` 可能是：**业务规则** / **防御性兜底** / **遗留 bug**。三者在代码里长得一样。

- 能从 flow-map 业务语义 + 命名 + 上下文确认是业务意图 → 写为 INV，标 `[码]`（+ `[runtime]` 若运行时也命中）。
- 像技术防御（空指针、越界、超时兜底）或意图不明 → **不写成业务 INV**；若值得记，标 `~inferred: 疑似技术兜底/意图待确认`。
- **永不**把"代码这么写"直接等同于"业务应该这样"。本产物只说「系统当前强制 X」，是事实陈述，不是规范背书。

## 5. 溯源与置信标注（每项必带）

**溯源锚（强制）**：代码位置 `类.方法` / `文件:行`，或 flow-map 节点号。无锚不写入。

**来源标（正交）**：
- `[码]` — 代码直读确认（第一依据）。
- `[runtime]` — 仅 flow-map 运行时证据见到，代码未深入确认。
- `[码][runtime]` — 双证，强确证。

**置信标（互斥）**：
- 确证 — 代码直读到，不标。
- `~inferred: <原因>` — 跨证据推断：①运行时未覆盖、仅代码可见的分支；②意图推测；③业务语言翻译不确定；④疑似防御兜底/遗留逻辑。

> 单样本盲区由代码补全是本 skill 价值所在：枚举给出**完整**状态机，但运行时只跑到部分迁移——未跑到的迁移标 `[码]`（提示"代码可见、未经运行验证"），不标 `~inferred`（它是代码确证的，只是没跑到）。

## 6. 多仓 targeting

CodeGraph 带 `projectPath=<前端|后端绝对路径>`；GitNexus 带 `repo=<仓名>`。前端查前端索引、后端查后端索引，不混用。

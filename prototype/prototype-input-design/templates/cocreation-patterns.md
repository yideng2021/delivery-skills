# 共创补全交互 · 五种模板

> 阶段 3 当输入不完整时使用。根据"缺失什么"选择合适的策略。

## 策略速查

| 缺失信息类型 | 推荐策略 | 说明 |
|-------------|---------|------|
| 操作流程不明 | 选择题式 / 示例引导式 | 提供常见流程模式供选择 |
| 数据对象与关系不全 | 预填确认式 | AI 推荐业务对象及属性列表，用户增删改 |
| 页面间关系不清 | 示例引导式 | 展示同类系统的页面关系图供参考 |
| 新功能不知放哪 | 定位辅助式 | 展示现有结构 + 多个放置位置选项 |
| 菜单结构缺失 | 预填确认式 | AI 基于功能清单推断菜单结构 |
| 角色权限不明 | 选择题式 | 列出常见的权限模式供选择 |
| 审核/审批流程不确定 | 示例引导式 | 展示简单/多级/会签等模式 |
| 界面打开方式不明 | 界面集成确认式 | 新页面/抽屉/弹窗/Tab 内嵌 |
| 界面有特殊集成要求 | 界面集成确认式 | iframe/微前端/嵌入第三方 |
| 视觉风格无偏好 | 多方案对比（策略二） | 提供 2-3 种风格方案 |
| 已有原型需对齐 | 定位辅助式 | 基于已有原型提取规范 |

## 模板一：选择题式

**适用**：有明确选项的场景（流程走向、权限模式、布局风格）。

```
📋 [要确认的主题]：

[问题陈述]？
  A. [选项1描述]
  B. [选项2描述]
  C. [选项3描述]

请选择（可多选，如 A 或 B+C）：
```

**示例**：
```
📋 操作流程需要确认：

任务创建完成后，下一步流向哪里？
  A. 直接生效，返回列表页（无审核流程）
  B. 提交审核，审核通过后生效
  C. 暂存为草稿，可稍后继续编辑

请选择（可多选）：
```

## 模板二：预填确认式

**适用**：AI 可合理推断的场景（字段列表、菜单结构、常见功能）。

```
📋 [要确认的主题]：

基于 [领域名] 的通用实践，AI 为 [页面/模块] 推荐以下 [字段/功能/项]：

| [维度] | 说明 | 来源 |
|--------|------|------|
| 项1 | xxx | 用户提供 |
| 项2 | xxx | AI推断-通用实践 |
| 项3 | xxx | AI推断-基于流程 |

请确认：
  ✅ 以上 OK，直接使用
  ➕ 需要增加：___
  ➖ 需要删除：___
  ✏️ 需要修改：___
```

## 模板三：示例引导式

**适用**：用户不确定如何描述的场景（流程、页面关系、审核模式）。

```
📋 [要确认的主题]：

以下是类似系统的常见 [流程/结构]，哪种更接近你的需求？

【方案A：[风格名]模式】
[流程/结构描述]

【方案B：[风格名]模式】
[流程/结构描述]

【方案C：其他】
请简要描述你的想法：___
```

**示例**：
```
📋 页面间关系需要确认：

【方案A：简单模式】
列表 → 新建 → 录入 → 保存 → 返回列表

【方案B：带审核模式】
列表 → 新建 → 录入 → 提交审核
                        ↓
                    审核列表 → 审核详情 → 通过/退回
                        ↓
                    生效完成 → 列表可见

【方案C：其他】
请简要描述：___
```

## 模板四：定位辅助式

**适用**：新功能不知放在哪的场景。

```
📋 新功能放置位置确认：

你提到需要新增"[功能名]"功能，以下是可能的放置位置：

【位置A】[位置描述]
  └── [做法说明]
  └── 优势：[优势]

【位置B】[位置描述]
  └── [做法说明]
  └── 优势：[优势]

【位置C】其他位置
  └── 请描述你的想法：___

参考：当前已有的菜单结构为：
  [展示现有结构]
```

## 模板五：界面集成确认式

**适用**：页面打开方式或集成要求不明的场景。

```
📋 界面集成方式确认：

以下页面需要确认打开方式和集成位置：

1. [页面名]，从 [入口] 点击后如何打开？
   A. 跳转新页面（信息量大、需独立操作）
   B. 右侧抽屉滑出（快速预览，不离开当前页）
   C. 弹窗打开（轻量信息，快速关闭）
   D. 其他：___

2. [另一页面]，从哪个入口进入？
   A. [选项1]
   B. [选项2]
   C. 其他：___

3. 是否有特殊集成要求？
   A. 无特殊要求，标准页面即可
   B. 需要嵌入到已有系统的某个位置（请说明：___）
   C. 需要以 iframe 方式嵌入第三方页面
   D. 其他：___
```

## 共创循环规则

```
共创补全 → 用户回答 → AI 更新输入原型设计文档 → 重新诊断
    ↑                                        │
    └──── 仍有关键信息缺失时继续循环 ────────┘
```

**循环终止条件**（任一满足即终止）：
1. 最小输入条件达标 → 进入阶段 4 生成
2. 用户主动要求"先生成看看" → 进入阶段 4，但生成结果必须标注信息不足的风险点

**单次补全的数量控制**：
- 一次最多提 **3 个问题**，避免用户疲劳
- 优先问必填要素（模块结构 / 页面功能 / 数据对象与关系 / 操作流程）
- 一次补全应聚焦**同一层级**的问题，不要跨层级混问

**"其他"选项处理**：
- 每种模板都保留"其他"选项让用户自由输入
- 用户选"其他"后，AI 应解析其输入并据此更新输入原型设计文档，而不是再发一轮问题

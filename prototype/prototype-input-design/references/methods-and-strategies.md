# 五种基础方式 + 两种增强策略

> 每次原型设计选一种**主基础方式**为主线（按输入形态划分，彼此正交），按需叠加**增强策略**。

## 一、五种基础方式

| 方式 | 输入形态 | 核心理念 |
|------|---------|---------|
| 方式一：视觉参考驱动 | 已有界面图/设计稿 | 以视觉为锚点复刻并微调 |
| 方式二：需求描述驱动 | 功能/规则文字 | 以功能为锚点自主设计 |
| 方式三：信息架构驱动 | 系统结构树 | 以架构为锚点逐页设计 |
| 方式四：对话渐进式 | 口述意图 | 通过对话逐步明确 |
| 方式五：存量迭代式 | 已有原型 | 基于上一版增量修改 |

### 方式一：视觉参考驱动
- **输入**：竞品截图、设计稿、手绘草图 + 差异调整说明
- **操作关键**：标注参考图关键区域，明确差异点
- **适用**：有明确竞品或参考设计、需要快速复刻并微调、客户给了参考截图

### 方式二：需求描述驱动
- **输入**：功能需求清单 + 业务规则 + 数据对象与关系 + 操作流程
- **操作关键**：先定义数据对象和关系，再说功能
- **适用**：需求明确但无视觉参考、希望 AI 发挥创作、功能导向型后台页面

### 方式三：信息架构驱动
- **输入**：模块树 + 页面清单 + 导航结构 + 数据模型
- **操作关键**：先搭骨架再填内容，按模块优先级分批生成
- **适用**：中大型系统整体设计、多模块协同的企业应用

### 方式四：对话渐进式
- **输入**：初始意图（一两句口述）
- **操作关键**：AI 主动提澄清问题，每轮收敛一点
- **适用**：需求模糊/探索阶段、非技术背景用户、快速头脑风暴

### 方式五：存量迭代式
- **输入**：现有原型 + 变更清单（[新增]/[修改]/[删除]/[优化]）
- **操作关键**：明确"未提及的部分保持不变"，差异对比后回归验证
- **适用**：客户确认后的需求微调、多轮迭代中的持续优化、版本升级

## 二、两种增强策略（可叠加在任一基础方式上）

| 策略 | 叠加效果 | 何时叠加 |
|------|---------|---------|
| 策略一：用户故事视角 | 为基础方式注入"角色-目标-场景"维度 | 多角色业务、注重体验、有敏捷故事输入 |
| 策略二：多方案对比择优 | 同需求并行输出 2-3 方案，人工择优 | 关键页面决策、客户提案、团队分歧 |

### 策略一：用户故事视角
在基础方式的提示词中附加：

```
【用户角色】- [角色] : [特征/诉求]
【用户故事】
1. 作为 X，我希望 Y，以便 Z
   - Given ... When ... Then ...
```

### 策略二：多方案对比
在基础方式的提示词中附加：

```
请提供 N（2-3）个不同风格的设计方案：
【方案差异维度】
- 方案 A：xxx 型 —— 适用 xxx
- 方案 B：yyy 型 —— 适用 yyy
- 方案 C：zzz 型 —— 适用 zzz
每个方案注明其适用场景和优劣势。
```

## 三、输入特征 → 方式 + 策略 的自动匹配

AI 在阶段 3 诊断时按下表匹配，并**向用户声明推荐理由**：

| 输入特征 | 推荐主基础方式 | 推荐叠加策略 |
|---------|---------------|-------------|
| 有参考图 + 有功能清单 | 方式一（视觉参考） | 无 |
| 有菜单结构 + 有业务流程 | 方式三（信息架构） | 无 |
| 有参考图 + 有多角色描述 | 方式一（视觉参考） | 策略一（用户故事视角） |
| 有功能清单 + 有多角色描述 | 方式二（需求描述） | 策略一（用户故事视角） |
| 有已有原型 + 有变更清单 | 方式五（存量迭代） | 无 |
| 只有口述想法 | 方式四（对话渐进） | 无 |
| 关键页面方向不确定 | 按主输入选 | 策略二（多方案对比） |
| 多角色 + 关键决策并存 | 按主输入选 | 策略一 + 策略二 |
| 视觉风格无偏好 | 按主输入选 | 策略二（多方案对比，给 2-3 风格让用户选） |

## 四、方式综合对比

| 对比维度 | 方式一 | 方式二 | 方式三 | 方式四 | 方式五 |
|---------|:---:|:---:|:---:|:---:|:---:|
| 输入准备门槛 | 低 | 中 | 高 | 最低 | 低 |
| AI 创作自由度 | 低 | 高 | 中 | 高 | 低 |
| 首次输出准确度 | 高 | 中 | 高 | 低 | 高 |
| 迭代效率 | 高 | 中 | 中低 | 中 | 最高 |
| 系统完整性 | 低（单页） | 中 | 最高 | 低 | 高 |
| 适合阶段 | 设计期 | 设计期 | 启动/设计期 | 探索期 | 迭代期 |

## 五、按项目阶段的组合建议

- **项目启动期**：方式四（对话渐进）探索需求 → 方式三（信息架构）搭建骨架
- **核心设计期**：方式一/二逐页设计 + 策略二（多方案对比）敲定关键页
- **迭代优化期**：方式五（存量迭代）持续微调 + 策略一（用户故事）走查

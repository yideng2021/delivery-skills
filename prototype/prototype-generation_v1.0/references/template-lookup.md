# 模板查找（template-lookup）

> **作用**：`prototype-generation` skill 阶段 4 启动时的入口规程——定位模板索引、描述匹配流程、规范未就绪/无匹配时的挂起输出。
> **本 skill 对模板的认知边界**：只读 `docs/prds/_templates.md` 索引；只把 `docs/prds/_template-<类型>/prototype/` 当作生成参考的"可视化样板"。不创建、不修改、不人机共创。

---

## 一、索引位置与形态

**固定位置**：`docs/prds/_templates.md`

**形态**：Markdown 文档，包含一张"模板注册表"——每行登记一个模板模块（`_template-<类型>/`）的基本信息与匹配信号。

**预期结构**（由上层策展 skill 维护，本 skill 只读解析）：

```markdown
# 项目模板索引

| 模板 ID | 形态 | 风格标签 | 覆盖页面类型 | 视觉提示 | 模块路径 | 状态 |
|---------|------|---------|------------|---------|---------|------|
| pc-admin-standard | PC 后台 | 商务蓝、深色侧栏、列表密集 | list / detail / form | 商务风、稳重、蓝色主调 | `_template-pc-admin-standard/` | ✅ 可用 |
| mobile-h5-light  | 手机 H5 | 浅色、卡片式、图文并重     | list / detail       | 清新、扁平、白底       | `_template-mobile-h5-light/`  | ✅ 可用 |
```

字段语义：
- **形态**（form-factor）：PC 后台 / 手机 H5 / 数据大屏 / 平板 / 其他
- **风格标签**（style-tags）：逗号分隔关键词，刻画视觉个性
- **覆盖页面类型**（covered-page-types）：该模板的 `prototype/pages/` 下已有的页面骨架类型
- **视觉提示**（visual-hints）：一句话描述，用于与 `input-design.md` "视觉参考"节对齐
- **模块路径**：相对于 `docs/prds/` 的路径
- **状态**：✅ 可用 / 🛠 建设中 / ⏸ 废弃

---

## 二、发现流程（阶段 4 启动时执行）

1. 打开 `docs/prds/_templates.md`
2. 解析注册表，提取所有 `状态 = ✅ 可用` 的行
3. 对每个候选读其 `模块路径` 下的 `input-design.md`（若存在）获得更完整的匹配元信息（模板自身作为"模块"也有自己的 input-design.md，描述了 typical-scenarios 等）
4. 合并本模块 `input-design.md` "视觉参考"+"页面功能"节信息 → 执行匹配评估

---

## 三、三种分支（互斥）

| 分支 | 判定条件 | 动作 |
|------|---------|------|
| **未就绪** | `_templates.md` 文件不存在；或文件存在但注册表为空；或所有条目状态非 `✅ 可用` | 输出"模板未就绪"挂起文本，结束本轮 |
| **无匹配** | 注册表有可用条目，但 AI 评估后所有候选匹配度均为"低"；或用户否决所有候选 | 输出"无匹配"挂起文本，结束本轮 |
| **有匹配** | 存在匹配度为"高"或"中"的候选，且用户采纳 | 进入生成（用模板模块的 `prototype/pages/` 作参考） |

**本 skill 不主动调用模板策展 skill**——挂起后由上层智能体决定是否调度。

---

## 四、匹配信号（按优先级）

1. `input-design.md` "七、视觉参考"节 vs 模板注册表的 `视觉提示` + 模板自身 `input-design.md` 的视觉参考节
2. `input-design.md` "二、页面功能"节的页面类型分布 vs 模板的 `覆盖页面类型`
3. `input-design.md` "一、模块结构"节的业务性质 vs 模板的 `typical-scenarios`（读自模板自身 input-design.md）
4. 本模块的 `form-factor`（从视觉参考或用户明示推断） vs 模板的 `形态`——**形态不符直接降为"低"**

**不设硬阈值**：匹配度高低由 AI 给出主观判断（高 / 中 / 低 + 文字理由），由用户决定是否接受。

---

## 五、向用户确认的输出格式

```
基于本模块的 input-design.md，从 _templates.md 中识别候选：

【首选】pc-admin-standard（匹配度：高）
  路径：docs/prds/_template-pc-admin-standard/
  理由：
    - 形态匹配（本模块为 PC 后台；模板形态 PC 后台）
    - 视觉参考"商务风 + 深色菜单"与模板 "商务蓝、深色侧栏" 吻合
    - 页面类型分布（3 列表 / 2 详情 / 1 表单）被模板 covered-page-types 完全覆盖

【备选】mobile-h5-light（匹配度：低）
  理由：形态不符（本模块为 PC 后台，模板为手机 H5）

是否采用【pc-admin-standard】作为生成参考？
  ✅ 采用 —— 进入生成
  🔁 查看模板详情（打开 _template-pc-admin-standard/prototype/ 展示页面骨架）
  ❌ 都不合适 —— 输出"无匹配"挂起文本
```

## 六、挂起文本格式（未就绪 / 无匹配）

**重要**：挂起文本**直接输出给用户**，不加额外装饰（不要套用"让我..."等过渡语）。输出完立即结束本轮，等待上层调度。

### 6.1 模板未就绪

```
当前项目的模板索引 `docs/prds/_templates.md` 尚未就绪（文件不存在 / 为空 / 所有条目状态非"✅ 可用"）。
本 skill 依赖此索引进行风格基线匹配，无法在未就绪状态下生成原型。

建议：切换到模板策展 skill 完成项目模板库初始化。完成后回到本 skill 继续生成 <模块名> 的原型。

提示：模板库初始化是一次性工作。首次完成后，后续业务模块可直接匹配复用。
```

### 6.2 无匹配

```
已读取模板索引 `docs/prds/_templates.md`，当前候选均与本模块（<模块名>）的视觉参考 / 页面类型分布不契合：
  - <模板 id 1>：匹配度 低（理由：...）
  - <模板 id 2>：匹配度 低（理由：...）

本 skill 不创建新模板。建议：切换到模板策展 skill 新增一个契合 <模块名> 的模板（形态：<从 input-design 推断> / 页面类型：<从 input-design 统计>）。完成后回到本 skill。
```

## 七、避免重复做（判定规则）

| 触发点 | 条件 | 动作 |
|-------|------|-----|
| 模板匹配 | 本模块 `prototype/` 尚未生成（首轮） | 执行匹配 |
| 模板匹配 | 本模块 `prototype/` 已存在（迭代轮） | **跳过**，迭代直接使用已有资源 |
| 模板查看 | 同一模板在本次对话已展示过 | 不重复展示 |
| 复用 `_shared/` | `_shared/` 存在 | 直接复用，不重新初始化（初始化是策展 skill 的职责） |
| 切换模板 | 用户主动要求"换一种风格生成" | 视为超出本 skill 职责，转交模板策展 skill |

**核心原则**：本 skill 不囤积、不缓存、不自建；每次都以 `_templates.md` 文件状态为准。

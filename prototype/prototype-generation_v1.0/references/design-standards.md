# 界面设计规范（视觉与布局）

> **适用产出形态**：PC 端 Web 高保真 HTML 原型。
> **使用方式**：原型生成时自动注入本规范。
> **可覆盖性**：以下为默认推荐值，项目有自己的 UI 规范时**用项目规范替换本文对应条目**。

---

## 一、全局三维布局

系统整体采用"顶部 Header + 左侧菜单 + 右侧内容区"的三维布局。

### 1.1 顶部 Header 栏

| 规范项 | 默认值 |
|--------|-------|
| 高度 | **60px**（固定） |
| 背景色 | 纯白 `#ffffff` 或深色品牌色；底部加浅色边框或 `box-shadow` 过渡 |
| 左侧 | 品牌 Logo + 系统名（20px / 加粗） |
| 右侧 | 用户信息 / 通知 / 设置 / 帮助 / 退出 |
| 交互 | 快捷图标 Hover 有颜色渐变或背景色变化 |

### 1.2 左侧菜单

| 规范项 | 默认值 |
|--------|-------|
| 宽度 | **展开 240px / 折叠 64px** |
| 背景色 | 深色（如 `#1e3a8a`）；文字与图标用白色或浅灰 |
| 选中态 | 背景加深或侧边高亮条 |
| 交互 | 支持层级折叠展开；顶部提供折叠/展开控制按钮；Hover 有视觉反馈 |

### 1.3 右侧内容区

| 规范项 | 默认值 |
|--------|-------|
| 宽度 | 占剩余空间（`flex:1` 或 `1fr`），自适应浏览器宽度 |
| 背景色 | 浅灰 `#f5f5f5`（衬托内部白卡片产生层次感） |
| 内部 padding | 16-24px |

## 二、内容区分区规范

### 2.1 面包屑

| 规范项 | 默认值 |
|--------|-------|
| 位置 | 紧贴顶部栏下方 |
| 高度 | 30-40px |
| 分隔符 | `/` 或 `>` |
| 文本色 | 非当前层级 `#6b7280`；当前页 `#1f2937`（加粗） |
| 交互 | 非当前层级加超链接，Hover 变色提示可点 |

### 2.2 查询区

| 规范项 | 默认值 |
|--------|-------|
| 容器 | 浅灰底 `#F5F7FA` + 边框 `1px solid #E8EAED`；与下方列表形成物理层级区隔 |
| 布局 | **4 列等宽栅格**（`minmax(0, 1fr)`）：**3 字段 + 1 按钮组**，按钮组始终占末格 |
| 响应式 | ≥1440px **4 列**；≤1200px **2 列** |
| 高低频分离 | 默认仅显示 2~3 个最高频字段 + 按钮组；其余字段通过"展开/收起"切换 |
| Label | **顶部对齐**（label 在上，控件在下）；14px |
| 间距 | 列间距 **20px**，行间距 **16px** |
| 输入控件 | 统一高度 **34px**；Input 含 × 清除图标；Select 选项 >7 内嵌搜索；多选以 Tag 展示（`...+N`） |
| 日期范围 | 单区间控件 `YYYY-MM-DD 至 YYYY-MM-DD`；快捷项（近7天/近30天/本月）与 label 同行右端排列 |
| 按钮组 | **查询**（主色实心，回车触发） / **重置**（线框，仅清空） / **展开/收起**（文本链接 + 箭头） |

### 2.3 列表区

| 规范项 | 默认值 |
|--------|-------|
| 容器 | 白底卡片；与查询区间距 16-24px |
| 表头操作区 | 左：标题/统计信息；右：全局操作（新增/批量导入/导出） |
| 表头样式 | 浅色背景 `#f9fafb`；字体加粗；下边框分隔 |
| 内容行 | 单元格 padding **12px 16px**；底边框分隔；**行 Hover 高亮** |
| 列对齐 | 文本左对齐；金额/数字右对齐；操作列居中 |
| 状态列 | 配色彩徽标或小图标（绿/黄/红灯） |
| 操作入口 | 文字按钮或图标按钮，主题色；**操作项 >3 个用"更多"下拉** |
| 分页 | 表格下方右对齐；含总条数 + 页码 + 上/下一页；当前页高亮 |

### 2.4 表单区

| 规范项 | 默认值 |
|--------|-------|
| 分主题展示 | 复杂表单必须分组；每组用独立卡片或带标题的区域（如"基础信息""扩展配置"） |
| 动态嵌套 | 嵌套规则用边框或左侧引导线包裹；子规则缩进；含 AND/OR 切换器 |
| 必填项 | Label 旁红色 `*` |
| 文本输入 | 标准 Input |
| 长文本 | Textarea 最小高度 80px，允许垂直拉伸；显示态保留换行符 |
| 文件上传 | 清晰的上传按钮或拖拽区；已上传展示文件名/大小/删除 |

### 2.5 Tab 页签

| 规范项 | 默认值 |
|--------|-------|
| 容器 | 顶部导航条，底色浅灰或与卡片头一致 |
| 页签项 | 默认中性色；**选中：文字加深加粗 + 底部品牌色高亮线** |
| 徽标 | 页签文字旁小圆角数字标签（Badge） |
| 子页签 | 比主页签更轻量（无边框纯文字样式，选中后文字变色） |

## 三、视觉系统

### 3.1 色彩体系

| 类型 | 用途 | 默认值 |
|------|------|-------|
| 主色 | 主按钮、选中状态、链接 | `#1e3a8a`（深蓝） |
| 成功 | 成功反馈 | `#10b981` |
| 警告 | 警告提示 | `#f59e0b` |
| 危险/错误 | 删除、错误 | `#ef4444` |
| 文本-标题 | 主要文本 | `#1f2937` |
| 文本-正文 | 正文 | `#4E5969` |
| 文本-辅助 | 辅助说明 | `#6b7280` |
| 文本-禁用 | 禁用态 | `#C9CDD4` |
| 背景-页面 | 内容区底 | `#f5f5f5` |
| 背景-表头 | 表头底 | `#f9fafb` |
| 边框 | 输入框/分割线 | `#d1d5db` 或 `#E5E6EB` |

### 3.2 字体排版

| 规范项 | 默认值 |
|--------|-------|
| 字体栈 | `'Microsoft YaHei', 'PingFang SC', sans-serif` |
| 大标题 | 20px / Bold |
| 标题 | 16px / Medium |
| 正文 | 14px / Regular |
| 辅助文字 | 12px / Regular |
| 行高 | 1.5 倍 |

### 3.3 圆角与阴影

| 规范项 | 默认值 |
|--------|-------|
| 圆角 | 按钮/输入框 **4px**；卡片 **6px** |
| 卡片阴影 | `0 2px 8px rgba(0,0,0,0.08)` |
| 图标 | 线性为主；笔画粗细 2px 统一；尺寸 16/20/24/32px |

### 3.4 间距与栅格

| 规范项 | 默认值 |
|--------|-------|
| 基础单位 | 8px |
| 常用间距 | 8/12/16/24/32/48px |
| 模块间距 | 32-48px |
| 元素间距 | 8-16px |
| 栅格 | 24 栏；列间距 16-24px |

## 四、状态反馈

| 状态 | 默认处理 |
|------|---------|
| 空态 | "暂无数据"插画 + 提示文案 + 引导操作按钮 |
| 加载中 | **骨架屏（Skeleton）**或 Spin；避免瞬间空白 |
| 操作反馈（写操作） | **Toast 成功/失败提示**（3 秒自动消失） |
| 危险操作 | 删除/提交需**二次确认弹窗** |
| 校验失败 | 输入框标红 + 下方红色提示文案 |
| 网络异常 | 显示重试按钮 |
| 权限不足 | 显示提示页面 |
| 按钮提交中 | **主按钮进入 Loading + Disabled 状态** |

## 五、按页面类型补充

### 列表页
- 顶部：筛选区 + 批量操作区
- 中部：数据表格（支持排序/列显示控制）
- 底部：分页器
- 空态：插图 + "暂无数据" + 引导按钮

### 表单页
- 字段分组，每组间距 32px
- 必填项红色星号
- 校验时机：失焦校验 + 提交校验
- 底部：取消 + 提交按钮（主按钮在右）

### 详情页
- 顶部：标题 + 关键状态标签 + 操作按钮
- 中部：信息分区（基本信息 / 关联数据 / 操作日志）
- 可用 Tab 切换大块内容

### Dashboard
- 顶部：时间范围筛选 + 全局指标
- 中部：图表矩阵（柱/折/饼/数字卡片）
- 支持下钻查看明细

## 六、规范注入方法

生成原型时在提示词中附加其一：

**A. 结构化精简提示词内嵌**（推荐，适合规范稳定场景）
```
【设计规范】
- 全局：顶栏 60px / 左菜单 240-64px / 右侧 #f5f5f5
- 内容区：面包屑 / 卡片式查询区 / 列表区 / Tab 页签
- 视觉：主色 #1e3a8a / 圆角 4-6px / 字体 YaHei+PingFang
```

**B. 文档引用**（适合规范内容丰富场景）
```
【设计规范】遵循 @references/design-standards.md
```

**C. 示例组件约束**（适合视觉还原精度要求高场景）
```
【设计规范】参考已有组件：{附截图或代码片段}
```

## 七、项目定制替换指南

当项目有自己的 UI 规范时，建议按以下方式覆盖默认值：

1. **准备项目规范文档**（如 `project-ui-standards.md`）
2. 在生成提示词中**同时引用两份规范**，并声明优先级：
   ```
   【设计规范】
   基础规范：@references/design-standards.md
   项目规范（优先）：@project-ui-standards.md
   冲突时以项目规范为准。
   ```
3. 或直接**用项目规范文档替换本文** `references/design-standards.md` —— 更彻底但需要确保覆盖完整。

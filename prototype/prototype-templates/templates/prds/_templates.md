# 原型模板注册表

> 本文件是 **prototype-generation skill 的唯一输入**。skill 在决定"是否复用已有模板"时，仅扫描此文件。
>
> 维护约定：
> - 每一行代表一个可复用的模板模块，指向 `_template-<类型>/` 目录。
> - 新增、修改、废弃模板时同步更新此表。
> - 列 `form-factor` / `style-tags` / `covered-page-types` 用于与需求输入做匹配；详细设计见各模板目录下的 `input-design.md`。

## 匹配信号速查

prototype-generation skill 读取需求文档时，按下列信号判定是否命中某模板：

### `pc-admin-standard` 命中信号

- 需求中出现 **"后台系统 / 管理系统 / 管理端 / B 端 / 内部系统 / 运营后台"** 任一表述；
- 页面形态以 **列表 + 详情 + 配置** 三类为主；
- 目标用户为 **企业员工 / 管理员 / 运营人员**（非面向 C 端消费者）；
- 视觉要求为 **信息密度高、严肃稳重、可复用企业配色**。

如所有信号都不命中，skill 应向用户确认是否新增模板，而非强行套用。

## 未命中时的处理

1. skill 输出提示："当前需求未命中任何已登记模板，建议：(a) 人工介入补充模板；(b) 暂按无模板生成"。
2. 不在此文件私自登记新模板；新增模板流程由模板策展 skill 或人工负责。

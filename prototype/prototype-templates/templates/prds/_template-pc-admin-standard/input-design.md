## 形态定位

| 维度 | 取值 |
| --- | --- |
| form-factor | PC / 桌面后台（最小支持宽度 1280px；响应式降级到 768px） |
| 主色调 | 深蓝 `#1e3a8a`（企业稳重风） |

| 字体 | 系统默认无衬线，中文优先（PingFang SC / Microsoft YaHei） |

## 覆盖的页面类型

1. **列表页**（`pages/list.html`）
   - 结构：顶部筛选区（关键字 + 2-3 个下拉筛选 + 查询/重置） + 标题 + 批量操作按钮 + 数据表格 + 分页
   - 适用：所有"查询 + 表格展示"的场景
2. **详情页**（`pages/detail.html`）
   - 结构：面包屑 + 页面标题 + 基础信息卡（info-grid）+ 指标评估卡（4 盏灯 card）+ 多 Tab 容器（支持子 Tab 嵌套）
   - 适用：需要展示复杂对象的综合情况、同时需要分维度浏览的场景
3. **表单配置页**（`pages/form.html`）
   - 结构：面包屑 + 页面操作区（标题 + 保存/取消）+ 多个分区卡片 + 动态行（可增减）+ 标签多选 + 底部重复操作区
   - 适用：多段式配置、后台规则设置、岗位/产品/策略配置

## 匹配信号（复用判定）

skill 通过 `_templates.md` 中登记的信号决定是否复用。以下任一条明显命中即可：

- 需求提到 **"后台系统 / 管理系统 / 管理端 / B 端 / 内部系统 / 运营后台 / 合规风控后台"**
- 需求目标用户是 **企业员工 / 管理员 / 运营人员 / 审核人员**（非 C 端消费者）
- 需求页面形态集中在 **列表 + 详情 + 配置** 三类
- 视觉要求 **信息密度高、严肃稳重、企业配色**

## 不适用场景

- **C 端消费者界面**（电商、社交、内容产品）：视觉偏活泼、信息密度低，与本模板不符
- **移动端 H5 / 原生 App**：布局基于 PC grid，不适配小屏主导的场景
- **可视化大屏**：本模板不含大屏布局规范，需单独建模板
- **轻量工具类落地页**：本模板是重型后台结构，会显得过度设计

## 复用步骤（给 skill 的提示）

1. 拷贝整个 `_template-pc-admin-standard/` 目录为 `<新业务模块名>/`
2. 按新业务改写以下内容：
   - `prototype/mock/*.js` —— 替换为新业务的模拟数据结构
   - `prototype/js/services/*.js` —— 字段名、接口签名匹配新业务
   - `prototype/pages/*.html` —— 文案、字段、Tab 结构按需裁剪
   - `prototype/css/pages/*.css` —— 如有特殊视觉诉求再扩展
3. 在 `_shared/reference/menu-data.js` 中登记新模块菜单项（**硬性前置**）
4. 不要改动 `_shared/` 下任何文件；如需调整共享样式，先在共享层迭代日志中评估影响面

## 演示业务选型

当前模板的演示业务为 **员工管理**（员工列表、员工详情、岗位配置），理由：

- 员工、部门、岗位、职级等概念对所有行业用户都通用，认知门槛低
- 数据结构（列表项 + 多字段对象 + 多维度配置）能完整覆盖三类页面的典型形态
- 绩效评估、晋升通道等场景足够丰富，便于展示详情页/表单页的复杂度

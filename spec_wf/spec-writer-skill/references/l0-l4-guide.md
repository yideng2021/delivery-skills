# L0–L4 分层指南

> spec 文档分 5 层,**严格分层**,各层职责互不重叠。
> AC 三层口径定义见 [`../../shared/contracts/ac-vocabulary.md`](../../shared/contracts/ac-vocabulary.md),本文件**不复述**定义,仅讲分层职责与写作要点。

---

## §1 层级总览

| 层级 | 名称 | 是否承载 AC | 关键产出 | 边界 |
|------|------|-------------|----------|------|
| L0 | 业务上下文 | ❌(仅业务禁区参与 DoD 守卫) | 业务目标 / 关键场景 / 业务禁区 | 不写功能细节 |
| L1 | 用户故事 | ❌(显式不承担验收) | US-x | 不写 AC、不写实现 |
| L2 | 业务实体与规则 | ✅ INV-x 唯一来源 | INV-x 列表 | 业务语言纯净 |
| L3 | 行为规约 | ✅ AC-{req}-xx 唯一来源 | EARS + Gherkin | 不写 HTTP / SQL / 字段类型 |
| L4 | DoD | ❌(仅反向引用) | checkbox 清单 | **零新增验收** |

---

## §2 L0 业务上下文写作要点

- **业务目标**:一句话概括业务价值,可被非技术干系人理解
- **关键场景**:1–3 条典型用户场景,每条 1 句
- **业务禁区**:本能力**不得做**的事(如"不向未成年用户推送营销内容"),将作为 L4 DoD 的最后一条守卫

不要在 L0 写:功能列表 / 角色权限矩阵 / 数据流图 / 时序图。

---

## §3 L1 用户故事写作要点

- 句式:`作为 {角色},我希望 {目标},以便 {价值}`
- 每条 US 必须能在 L2 INV 或 L3 AC 中找到承载点(向下追溯无承载即视为悬空)
- US 编号 `US-1`、`US-2` ...,在 spec 文件内单调递增
- **不写 AC**:US 是叙事不是验收;真正可执行的验收在 L2/L3

不要把"系统应当 ……"写进 L1—— 那是 L2 INV 或 L3 AC 的范畴。

---

## §4 L2 业务实体与规则(INV-x)写作要点

- INV 描述**永真**的业务规则(与时间、操作、用户行为无关)
- 编号 `INV-1`、`INV-2` ...,在 spec 文件内单调递增
- 业务语言纯净 —— 不写以下技术词:
  - 数据库相关:`主键 / 外键 / 索引 / 表 / SQL / 1:N`
  - 实现相关:`聚合根 / 仓储 / 字段类型 / DTO`
  - 接口相关:`HTTP / REST / gRPC / API 路径`
- 必须**可验证**:可由静态数据快照判定真伪
- 不写"系统应当 ……"句式(那是 AC 的范畴)

INV 与 AC 的判别口诀:**数据级永真规则归 INV;触发后行为归 AC**。

---

## §5 L3 行为规约(AC-{req}-xx)写作要点

- 每条 AC = **EARS 句**(自然语言 shall 句) + **Gherkin Scenario**
- 编号 `AC-{req}-{seq}`:
  - `req` = `related_req` 元素尾段(例如 `related_req: [AUTH-signup-01]` 对应 `AC-signup-01`)
  - `seq` = 同一 `req` 下的场景序号
- EARS 句式见 [`./ears-gherkin-cheatsheet.md`](./ears-gherkin-cheatsheet.md)
- Then 子句必须**可观察、可断言**;禁止"运行流畅 / 体验良好"等模糊形容
- 不写 HTTP code / 字段名 / SQL / 测试命令

L3 不重复 L2 INV 的内容:若行为本质是数据级规则,归 L2。

---

## §6 L4 DoD 写作要点

- 形态:checkbox 清单
- **零新增验收铁律**:每条 checkbox 必须能在 L2 INV / L3 AC / L0 业务禁区中找到来源
- 工程闭环条目可统一为 1–2 条总结勾(类型检查 / lint / 构建 / L0 业务禁区守卫)
- 不写覆盖率指标(如 `≥ 80%`)、不写具体测试命令

DoD 是"勾选清单",不是"验收定义";验收定义只在 L2/L3。

---

## §7 跨层引用与追溯

每条 US(L1)→ 至少 1 条 INV(L2)或 1 条 AC(L3);
每条 INV(L2)/ AC(L3)→ 必须出现在 L4 DoD 的某条 checkbox 中。

追溯断链(US 找不到承载、INV/AC 未进 DoD、DoD 引用了不存在的编号)在 [`./checklist.md`](./checklist.md) §2 / §3 中被显式拦截。

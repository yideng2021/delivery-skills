# 低耦合 / 低影响设计规则目录(§5)

> 本文件是 `impact.md` §5「低耦合/低影响设计规则」的写法权威。SKILL.md 不复述本文条目。
>
> **定位铁律**:本节只给**通用软件工程设计原则 + 适用条件**,作为给 design / spec 的**参考**;
> **不**给针对当前代码块的具体落地方案——具体怎么落地由 design / spec 的 skill 设计。**分工明确,不越位。**

---

## §1 本节"给什么 / 不给什么"

| ✅ 给(诊断器职责) | ❌ 不给(design / spec / dev 职责) |
|--------------------|-----------------------------------|
| 命名的通用设计原则(Expand-Contract、ACL…) | 具体类名 / 方法签名 / 代码片段 |
| 该原则的适用条件(一句话) | 具体回滚开关 key / 配置项 |
| 推荐给哪个下游阶段参考 | 具体特征测试用例 / 断言 |
| 为什么这条原则降低本次的耦合/影响(一句话) | 本工程的目录结构 / 落地步骤 |

> 一旦本节出现"具体到本工程代码"的内容,即为越位(混入了 design 的活),应打回。

---

## §2 原则目录(按"冲突/影响类型 → 推荐原则"映射)

> 这是一份**参考目录**,非强制枚举、非 validator 校验项。分析器据 §2/§3 的条目类型,挑 1–2 条最贴切的原则填入 impact.md §5。

| 冲突/影响类型 | 推荐通用原则 | 适用条件(一句话) | 主要参考阶段 |
|--------------|-------------|-------------------|-------------|
| **既有接口变更** | Expand-Contract(并行新旧,先扩后缩) | 调用方多、需平滑过渡 | design 模块契约 / spec 迁移 |
| 既有接口变更 | 适配器(Adapter)隔离 | 新旧签名需并行一段时间 | design 模块契约 |
| **既有逻辑替换** | Branch by Abstraction | 大块替换需可灰度、可回退 | design ADR |
| 既有逻辑替换 | 防腐层(ACL / Anti-Corruption Layer) | 新旧语义需隔离、防污染 | design BC 关系(ACL隔离) |
| **数据语义变更** | Additive-only schema(只增不改) | 不可破坏既有读路径 | spec 迁移路径 / design |
| 数据语义变更 | Expand-Contract 双写迁移 | 需在线迁移、零停机 | spec 迁移路径 |
| **跨模块波及** | Façade 收口 | 波及面广、需收敛接触点 | design BC/模块切分 |
| 跨模块波及 | 依赖倒置(DIP) | 高层不应依赖低层细节 | design 模块依赖图 |
| **跨团队所有权** | 接口先行 + 契约测试(契约层而非实现层耦合) | 跨 owner 协作 | design 模块契约 / spec AC |
| **行为兼容性风险** | 既有行为应被"钉住"(由下游补特征测试) | 触达既有方法 | spec L4 DoD / dev |

> 最后一行只**指出方向**(既有行为需被钉住),**不**在本 skill 登记或编写特征测试——那是 spec DoD / dev 的职责。

## §3 选用准则

1. **贴着 §2/§3 选**:每条规则必须能回溯到 §2 某条冲突或 §3 某条影响;不写"泛泛而谈"的原则。
2. **少而精**:每条冲突/影响最多配 1–2 条原则;不堆砌原则清单。
3. **只给原则名 + 适用条件**:落地细节留给 design/spec。
4. **不确定时**:宁可少给,也不臆造与本次无关的原则。

## §5 棕地禁止行为(硬禁,命中即视为越界)

来自原棕地约束的「禁止行为清单」,在本 skill 落地:

- ❌ **顺手优化**:本次意图外的代码风格 / 命名 / 结构调整
- ❌ **顺手升级依赖**:依赖版本变更必须独立提案
- ❌ **跨域格式化**:跨包 / 跨模块的批量 reformat 与本次改动混提
- ❌ **隐式破坏**:用反射 / monkey-patch / 字节码增强绕过封装(详见 [`invasion-and-seam.md`](./invasion-and-seam.md) §5)
- ❌ **越位写落地方案**:在 §5 写具体代码 / 回滚 key / 测试断言(那是 design/spec/dev 的活)
